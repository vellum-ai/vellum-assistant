- **Resume sessions** - on startup, display a list of all sessions ordered by last activity and allow the user to resume a session by selecting it, or create a new session.
- **Slash commands** — `/new`, `/switch <session>`, `/clear`, `/model <name>`, `/cost`, `/history`, `/undo`, `/compact`. Quick actions without leaving the chat.
- **Command injection hardening** — the shell tool passes commands through `bash -c`. Audit for edge cases where tool input could escape. Consider using `--` separators.
- **Dependency auditing** — `web-tree-sitter` and `tree-sitter-bash` are WASM binaries loaded at runtime. Pin exact versions and verify checksums.
- **Hot reload** — when the daemon is running and you change config or trust rules, pick up changes without restart.
- **Usage and cost tracking** — track input/output tokens per message, per session, cumulative
- **File write previews** — before writing a file, show the user a diff and ask for confirmation. Especially for overwrites.
- **Daemon health monitoring** — watchdog that restarts the daemon if it crashes. Heartbeat checks, memory usage monitoring.
- **Web search tool** — search the web and return results. Enables the agent to look things up when it doesn't know something. Uses Brave Search API.
- **Extended thinking** — support Claude's extended thinking mode for complex reasoning tasks. Pass thinking budget, relay thinking blocks to the client.
- **System prompt** — the default system prompt is minimal. Add SOUL.md and IDENTITY.md files in `~/.vellum/` that can be used to customize the system prompt. Look at OpenClaw's SOUL.md and IDENTITY.md for inspiration and for how they should be structured and used.
- **OpenAI models** - Implement the provider and support GPT-5.2, GPT-5 mini and GPT-5 nano.
- **Gemini models** - Implement the provider and support Gemini 3 Pro and Gemini 3 Flash.
- **Ollama/local models** — Used for privacy-sensitive work or offline use. Talk to a local Ollama instance. Implement the provider and support OpenAI-compatible mode: https://docs.ollama.com/api/openai-compatibility
- **Multi-line input** — support pasting or composing multi-line messages. cmd+enter to send.
- **Image/file input** — drag-and-drop or paste images, attach files to messages, support vision/files at model provider level. Allow files in the CLI too.
- **Tab completion** — autocomplete commands, file paths, session names. Arrow keys to navigate, Enter to select.
- **Encrypted API key storage** — config.json stores API keys in plaintext. Use the OS keychain (macOS Keychain, Linux secret-service) or at minimum encrypt at rest.
- **Integration tests for IPC** — test the full client-daemon message flow, reconnection, session switching, confirmation flow.
- **Agent loop tests** — mock the provider, test tool execution flow, error handling, max-turns behavior.
- **End-to-end tests** — spin up daemon, connect CLI, send messages, verify responses. Test the full stack.
- **Plugin system** — let users write custom tools as standalone scripts or modules. Drop a file in `~/.vellum/plugins/`, it auto-registers. Define a simple interface: name, description, input schema, execute function.
- **Hooks/events** — user-defined scripts that run on events (pre-tool-execute, post-message, on-error). Like git hooks but for the assistant.
- **Binary distribution** — compile the assistant to a standalone binary with `bun build --compile`. Distribute via Homebrew, curl install script, or GitHub releases.
- **Release automation** — version bumping, changelog generation, binary builds.
- **Rich terminal UI** — replace raw readline with something like Ink (React for CLI) or blessed-contrib. Markdown rendering, syntax-highlighted code blocks, progress spinners, split panes.
- **Context window management** — track token usage, auto-summarize/compact conversation history when approaching limits. Right now conversations just grow unbounded until they hit the API limit and error. Research SOTA techniques for context window management for conversations that can span over months or years and grow to hundreds of millions of tokens.
- **Improved memory management** — research SOTA techniques for agent/LLM memory management for conversations that can span over months or years and grow to hundreds of millions of tokens. Improve the memory system to support this. Memory should be stored in `~/.vellum/` and should be globally accessible to all sessions.
- **Multi-agent orchestration** — spawn sub-agents for parallel work. One agent plans, others execute. Useful for large refactors or multi-file changes.
- **Remote daemon** — connect to a daemon running on a remote machine over SSH tunnel or TLS socket. Useful for working on servers.
- **Live connection to local daemon** — the web UI should be able to connect to a running local daemon, not just manage cloud agents. WebSocket bridge.
- **Conversation viewer** — browse and search past conversations in the web UI. Currently only the CLI can access local conversations.
- **Trust rule editor** — manage trust rules from the web UI with a nice table view, add/remove/edit.
- **Usage dashboard** — token usage, cost breakdown, tool invocation stats over time. Charted.
- **Mobile-friendly** — responsive design for the web UI so you can check on agents from your phone.
