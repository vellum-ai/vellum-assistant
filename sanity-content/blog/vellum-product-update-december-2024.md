You can now specify whether an input variable is required vs. optional, and set default values for optional variables in both Prompts &amp; Workflows. ### Error Handling in Guardrail Nodes

The "Reject on Error" toggle enables you to catch and handle errors directly within Guardrail Nodes to ensure a smooth process execution. Previously there was no way to proceed if your Guardrail failed but with the new toggle, you can continue on without aborting the Workflow and handle the error yourself.
